# Playwright API & UI Testing Project

This repository contains a set of end-to-end tests using [Playwright](https://playwright.dev/) targeting both API and UI endpoints of a sample application. It is configured to run in Chromium and includes custom setup steps for authentication.

---

## 📁 Project Structure

```
playwright_api/                # project root
├─ package.json                # npm project configuration and test scripts
├─ playwright.config.js        # Playwright configuration file
├─ tests/                      # all Playwright test sources
│  ├─ authAPI.setup.js         # global fixture for API authentication
│  ├─ authUI.setup.js          # global fixture for UI authentication
│  ├─ test_1.spec.js           # example UI test with request interception
│  └─ test_2.spec.js           # API test demonstrating POST request
├─ .auth/                      # (generated) storageState files
│     ├─ loginAPI.json         # saved API auth state
│     └─ loginUI.json          # saved UI auth state
├─ test-results/               # output directory for test artifacts
└─ playwright-report/          # HTML report generated by `npm run report`
```

(The `.auth` folder is created automatically after running the authentication fixtures.)

---

## 📝 Description

This repository is designed for quality assurance engineers who want to run automated Playwright tests against a web application. It contains:

- **API tests** that use Playwright's `request` fixture to interact with REST endpoints.
- **UI tests** that automate Chromium (or other browsers) to exercise the user interface.
- **Authentication helpers** which login once and save the authenticated `storageState` to speed up subsequent tests.
- **Routing/stubbing** example showing how to intercept network requests and provide mocked responses.

The tests are written in JavaScript (CommonJS) and rely on environment variables for credentials.

## 🚀 Getting Started

1. **Clone the repository**
   ```bash
   git clone <repo-url>
   cd playwright_api
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Configure environment variables**
   Create a file named `.env` in the project root (ignore in git) with the following values:
   ```env
   TEST_USER=your_email@example.com
   TEST_PASS=yourPassword
   ```
   These credentials are used by the auth setup scripts.

4. **Run tests**
   - Execute all tests in headless mode:
     ```bash
     npm run test
     ```
   - Run tests with a visible browser window:
     ```bash
     npm run test:headed
     ```
   - Only run UI tests (Chromium, UI project):
     ```bash
     npm run test:ui
     ```
   - Only run API tests (Chromium, API project):
     ```bash
     npm run test:api
     ```
   - Perform authentication setup without running other specs:
     ```bash
     npm run setup
     ```

5. **View reports**
   After running tests, generate an HTML report:
   ```bash
   npm run report
   ```
   Then open `playwright-report/index.html` in your browser.

## ⚙️ Configuration Notes

- **`playwright.config.js`** contains project definitions (`chromium-ui`, `chromium-api`, `setup`) and global hooks.
- **Storage state files** (`.auth/loginUI.json` and `.auth/loginAPI.json`) are consumed by tests to pre-authenticate the browser or API context.
- **Environment management** relies on `dotenv` (already included) so `.env` values are loaded automatically when tests run.

---

## 🛠️ Extending the Suite

- Add new spec files under the `tests/` folder following Playwright's naming conventions (e.g. `*.spec.js`).
- Use the existing `authUI.setup.js` or `authAPI.setup.js` fixtures to share authentication state.
- Update `playwright.config.js` to include new projects or modify existing ones.
